## TRUE MiroFish vs Pretend MiroFish Discussion

**Time:** ~7:45 AM

### The Distinction
User asked critical question about how close our implementation was to actual MiroFish:
- **Pretend MiroFish:** 1 prompt, Kimi "acts like" 5 people → NOT independent (groupthink risk)
- **TRUE MiroFish:** 50 separate API calls, each agent isolated → REAL swarm intelligence

### TRUE MiroFish Implementation Attempt
**Script:** `true-mirofish-swarm.ts`
- Uses Groq API (llama-3.3-70b-versatile)
- 50 separate LLM calls per player
- Weighted consensus by confidence
- Cost: ~$0.03 per player, ~$0.60 full field

### Rate Limiting Issue Encountered
**Error:** 429 Rate limit exceeded on Groq free tier
- Limit: 12,000 tokens/minute
- Running 50 agents in parallel exceeded this
- Only partial results obtained:

```
📈 PARTIAL CONSENSUS RANKINGS
1. Scottie Scheffler    | 19.5%
2. Jon Rahm             | 17.1%
3. Rory McIlroy         | 16.8%
4. Xander Schauffele    | 15.0%
5. Hideki Matsuyama     | 9.4%
```

**Confidence scores low** (~0.3/10) because most agents failed to respond.

### Solutions Discussed
1. **Add rate limiting:** Batch requests (5 at a time) with delays
2. **Add retries:** Exponential backoff on 429 errors
3. **Reduce agent count:** Use 10-15 agents instead of 50
4. **Switch model:** Use mixtral-8x7b (higher rate limits)

### Key Learning
TRUE MiroFish requires independent API calls — "pretend" agents in one prompt don't provide genuine diversity of opinion. The statistical formula approach (from earlier) is actually more honest than fake multi-persona prompts.
