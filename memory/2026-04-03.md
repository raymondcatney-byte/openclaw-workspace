### Polymarket Market Feed - Discovery Solution

**Problem:** Tag-based API fetches (`?tag_id=100265`) return same 5-10 markets always. Not dynamic like website.

**Root Cause:** Gamma API `/events?tag_id=X` doesn't support sorting params (`sort=volume`, `sort=trending` all fail).

**Solution:** `api-market-curated-discovery.ts` - Category whitelist + rotating pagination

**Architecture:**
```
WHITELIST CATEGORIES:
- GEOPOLITICS (100265)
- ECONOMY (100328)
- FINANCE (120)
- TECH (1401)
- CRYPTO (21)
- POLITICS (2)

DISCOVERY OFFSETS: [0, 20, 40, 60, 80, 100]
Rotation: Hourly (based on current hour % offsets.length)
```

**How it works:**
1. Fetches each whitelisted category at current offset
2. Fetches second page (offset+20) for top 3 categories
3. Deduplicates, filters sports/entertainment, sorts by volume
4. Returns 30-50 markets that change every hour

**Files Created:**
- `api-market-curated-discovery.ts` - Final working solution
- `api-market-dynamic.ts` - Multi-source approach (all categories)
- `debug-polymarket.mjs` - API testing script
- `test-gamma-params.mjs` - Sorting parameter tests
- `test-trending-endpoint.mjs` - Endpoint discovery tests

**GitHub:** https://github.com/raymondcatney-byte/openclaw-workspace/blob/main/api-market-curated-discovery.ts

**Key Insight:** Polymarket website combines multiple data sources. No single API endpoint provides trending/dynamic results. Pagination offset is the only reliable discovery mechanism.
