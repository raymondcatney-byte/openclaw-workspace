# 2026-03-31 — Polymarket Bug Fixes & Contingency Architecture

## Context
Polymarket functions in dashboard failing on tag/slug resolution. 5 critical bugs identified causing incorrect results.

## Bugs Fixed

### Bug #1: Slug Normalization Mismatch
- **File:** [action].ts ~line 340
- **Issue:** Comparing encoded slugs against decoded API responses; case sensitivity issues
- **Fix:** Added `normalizeSlug()` helper with decodeURIComponent, lowercase, special char normalization

### Bug #2: Category Mapping Fails Silently
- **File:** polymarket_watchlist.ts ~line 280
- **Issue:** Categories not in WATCHLIST_CATEGORY_ALIASES get quarantined as "missing"
- **Fix:** Added fuzzy fallback matching (politic→GEOPOLITICS, tech→AI, finance→MACRO, etc.)

### Bug #3: Tag Filtering Uses Wrong Property
- **File:** polymarket_watchlist.ts ~line 185
- **Issue:** Checking `record.tags` twice instead of proper tag object extraction
- **Fix:** Handle both string arrays and object arrays ({slug, name}) from Polymarket API

### Bug #4: Search Scoring Skips Valid Matches
- **File:** [action].ts ~line 210
- **Issue:** Returns null if ANY field is missing, even when other fields have data
- **Fix:** Filter empty strings before fullText check, only null if truly empty

### Bug #5: Event-to-Market Slug Resolution Broken
- **File:** [action].ts ~line 400+
- **Issue:** `extractMarketIdFromPayload` returns candidates[0] regardless of slug match
- **Fix:** Find exact slug match first, only fallback if single candidate

## Contingency Architecture Built

Tiered fallback system for category-based Polymarket search:

| Tier | Source | Latency | Trigger |
|------|--------|---------|---------|
| 1 | Local Cache (Redis/Upstash) | <10ms | Primary |
| 2 | Gamma API Direct | ~200ms | Cache miss |
| 3 | Watchlist + Discovery Hybrid | ~300ms | API timeout/rate limit |
| 4 | Static Snapshot | <50ms | All APIs down |

**Key Components:**
- `PolymarketCategorySearch` class with resilience patterns
- Circuit breaker (5 failure threshold, 30s cooldown)
- Monitoring webhook for fallback tier alerts
- Zero UI changes — transparent resilience layer

**API Endpoint:** `GET /api/polymarket/category?category=GEOPOLITICS&freshness=balanced`
