## 2026-04-04 (Evening) — RTDS Market Data & Real Polymarket Integration

### L0 (Abstract)
Built complete RTDS (Real-Time Data Service) WebSocket integration for live market prices and connected real Polymarket Gamma API for live prediction market data. Sovereign signals now use actual divergence between prediction markets and live prices.

### L1 (Overview)
**RTDS WebSocket System:**
- Hook: `useRTDS.ts` - WebSocket connection to `wss://ws-live-data.polymarket.com`
- Auto-reconnect with exponential backoff (3s base, max 10 attempts)
- 5-second PING heartbeat to maintain connection
- 20 default symbols: equities (AAPL, TSLA, NVDA, MSFT, GOOGL, META, COIN, HOOD), ETFs (SPY, QQQ, VXX), crypto (BTC, ETH, SOL), forex (EURUSD, USDJPY), metals (XAUUSD, XAGUSD), commodities (WTI, NGD)
- Asset class categorization with price normalization
- Stale data detection via `is_carried_forward` flag

**Cross-Asset Signal Engine:**
- Hook: `useCrossAssetSignals.ts` - Regime detection across 10 regime types
- 7 signal types: regime_shift, divergence, confluence, momentum_surge, volatility_spike, correlation_break, cross_asset_lead
- Rolling window analysis (60s, 300s windows)
- Regime confirmation delay: 60s (avoid whipsaws)
- Signal cooldown: 300s (prevent spam)

**Unusual Movement Detection:**
- Hook: `useUnusualMovementDetection.ts` - Price anomaly detection

**Signal Generation Logic:**
1. Fetch live PM events (50 max, filtered by liquidity/volume)
2. For each event, extract related assets via keyword matching
3. For each related asset with active RTDS price:
   - Calculate alignment score between PM probability change and price change
   - Determine signal type based on divergence/confluence patterns
   - Generate thesis with time horizon and conviction level
   - Score edge (0-100) based on opportunity quality
4. Deduplicate by event+symbol pair
5. Sort by edge score descending
6. Emit high-edge signals (≥70) to callback
