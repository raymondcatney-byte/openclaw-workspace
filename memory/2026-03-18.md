
## March 18, 2026 — Globe Data Feed (Added)

### Problem
User wants to enhance globe visualization WITHOUT changing any other War Room UI.

### Solution
Created `globe-data-feed.ts` - a standalone module that:
- Transforms raw data into globe-ready format
- Supports 10 data layers: conflicts, vessels, flights, sentiment, infrastructure, cyber, ports, markets, earthquakes, simulations
- Provides visual properties (color, size, pulse, opacity) that globe consumes directly
- Includes moving object support (trails for vessels/flights)
- Supports heatmap regions (sentiment overlays)
- Supports animated arcs (trade routes)

### Data Structure
```typescript
interface GlobePoint {
  id, lat, lng, layer, timestamp
  color, size, opacity, pulse        // Visual
  title, description, severity       // Tooltip
  heading, speed, trail              // Moving objects
  radius, intensity                  // Heatmap
  makaveliQuery, bruceQuery          // Click actions
}
```

### Transformers
- `conflicts`: ACLED -> Red dots sized by fatalities, pulsing for 20+ deaths
- `vessels`: AIS -> Ship icons with 20-point trails, color by speed
- `flights`: ADS-B -> Aircraft icons, cyan for high altitude, red for emergencies
- `sentiment`: Telegram -> Regional heatmap (red=fearful, green=neutral)
- `infrastructure`: DNS -> Site status (green=up, red=down, pulse on outage)
- `earthquakes`: USGS -> Magnitude circles, radius scales with magnitude
- `ports`: Congestion -> Status rings, size by vessel count
- `tradeRoutes`: Volume arcs -> Animated flow lines

### Usage
```typescript
import { globeDataFeed } from './war-room';

// Subscribe (your globe calls this)
feed.subscribe(state => updateGlobe(state.points, state.arcs, state.regions));

// Inject from your existing systems
feed.injectPoints('conflicts', acledData);
feed.injectRegions('sentiment', telegramData);

// Handle clicks
const nearby = feed.getPointsNear(lat, lng, 50);
nearby.forEach(p => p.makaveliQuery && triggerMakaveli(p.makaveliQuery));
```
